## ⚙️ Getting Started

### 1️⃣ Fork the Repository

Click the **Fork** button at the top right of this repo.

### 2️⃣ Clone Your Fork

```bash
git clone https://github.com/nikhil-ai-insights/<your-repo-name>.git
cd <your-repo-name>
```

### 3️⃣ Create a New Branch

```bash
git checkout -b feature/your-feature-name
```

## 🛠️ Development Guidelines

- Keep workflows **clean and modular**
- Use **clear naming** for all nodes
- **Avoid breaking** existing functionality
- **Test your changes** thoroughly before submitting
- Follow **n8n workflow best practices**

---

## 🧪 Testing Your Changes

Before submitting a pull request, make sure:

- ✅ Workflow runs without errors
- ✅ Tested with sample Google Sheet data
- ✅ Email sending logic works as expected
- ✅ No duplicate emails are sent

---

## 📝 Commit Guidelines

Use clear, meaningful commit messages following this format:

```
feat: add AI email personalization
fix: resolve duplicate email issue
docs: update README
refactor: clean up IF node logic
```

| Prefix | When to Use |
|--------|-------------|
| `feat` | Adding a new feature |
| `fix` | Bug fix |
| `docs` | Documentation changes |
| `refactor` | Code/workflow restructuring |
| `test` | Adding or updating tests |

---

## 🚀 Submitting a Pull Request

**1.** Push your changes:

```bash
git push origin feature/your-feature-name
```

**2.** Go to the repo on GitHub and click **"Open Pull Request"**

**3.** In your PR description, clearly include:

- 📝 **What** you changed
- 💡 **Why** you changed it
- 📸 **Screenshots** (if applicable)
