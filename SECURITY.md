🔐 Security Policy
📌 Supported Versions

We actively maintain and support the latest version of this project.

Version	Supported
Latest	✅ Yes
Older Versions	❌ No
🚨 Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

📩 How to Report
Email: (nikhilkumarofficiall123@gmail.com)
Or open a private issue (if enabled)
Avoid creating public issues for sensitive vulnerabilities
🛡️ What to Include in Report

Please include the following details:

Description of the vulnerability
Steps to reproduce
Possible impact
Screenshots or proof-of-concept (if applicable)
Suggested fix (optional but appreciated)
⏱️ Response Time

We aim to:

Respond within 48–72 hours
Provide a fix or update within 7 days (depending on severity)
🔒 Security Best Practices

This project involves email automation and integrations. Please follow:

📧 Email Safety
Do not use for spam or malicious outreach
Always respect email regulations (GDPR, CAN-SPAM, etc.)
🔑 Credentials Protection
Never commit API keys or credentials
Use environment variables or n8n credentials manager
Rotate keys regularly
📊 Data Handling
Avoid storing sensitive personal data
Use minimal required information
Protect Google Sheets access
⚠️ Known Risks
Misuse of automation for spam
Exposure of email credentials
Improper rate limiting leading to account restrictions
🧰 Recommended Security Setup
Enable 2FA on:
GitHub
Gmail
Google Cloud
Use:
OAuth2 authentication
Secure API connections
Access-restricted Google Sheets
🚫 Out of Scope

The following are NOT considered security vulnerabilities:

Feature requests
UI/UX improvements
Minor bugs with no security impact
🙏 Responsible Disclosure

We appreciate responsible disclosure and will:

Acknowledge your contribution
Fix the issue as soon as possible
Keep communication transparent
📢 Disclaimer

This project is intended for ethical and professional use only.
Any misuse (spam, phishing, abuse) is strictly discouraged.
